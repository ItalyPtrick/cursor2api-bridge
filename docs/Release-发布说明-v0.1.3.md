# Cursor2API Bridge v0.1.3

这是一版以“首页控制台重构”为主的小版本更新。

## 这版改了什么

- 首页控制台改成了更紧凑的单主列工作台
- 服务控制、重启和 OCR 开关现在集中放在首页顶部
- 首页不再单独放“接入信息”大卡，改成顶部快速复制入口
- 详细接入信息移到 `设置` 页面集中查看
- 首页的“标准输出”区域明显放大，可以一眼看到当前缓存的 18 行
- 首页的“标准错误”改成底部窄栏，异常时更容易快速定位

## 你会感受到什么变化

### 首页更像真正的工作台

以前首页的信息块更分散。

从 `v0.1.3` 开始，首页主要聚焦三件事：

1. 启动 / 停止 / 重启服务
2. 切换 OCR
3. 直接看完整的最近标准输出

如果你平时主要是在首页“开服务、看输出、排错”，这版会更顺手。

### OCR 开关位置变了

以前 OCR 开关在 `设置` 页面。

从 `v0.1.3` 开始，它被放到了首页服务控制区，作为一个更靠近实际使用场景的附属开关。

也就是说：

- 想开关 OCR，不用再切去设置页
- 首页就能直接控制

### 详细接入信息还在，只是换了位置

首页现在只保留快速复制：

- 本地地址
- 当前端口
- 当前模型

更完整的接入与运行信息，比如：

- 健康检查地址
- Node 运行时路径
- 服务版本
- 应用版本
- 上游版本

都放到了 `设置` 页面里。

## 更新方式

这版没有改动更新流程。

### 软件内更新

1. 打开旧版本
2. 进入 `设置`
3. 点击 `检查更新`
4. 有新版本后点击 `下载并解压`
5. 下载完成后点击 `启动新版本`

### 手动更新

1. 下载新的 zip
2. 解压到新的目录
3. 如需保留配置、日志和数据库，把旧目录里的 `data/` 复制到新目录

## 下载说明

软件内更新依赖这两个资产：

- `cursor2api-bridge-portable-win-x64.zip`
- `cursor2api-bridge-portable-win-x64.zip.sha256`

`portable.exe` 只作为附加下载，不参与软件内更新。

## 小提醒

- 如果你想确认服务有没有正常跑起来，先看首页顶部状态条
- 如果服务有异常，先看首页底部的“标准错误”窄栏和顶部错误摘要
- 如果你需要完整接入细节，进入 `设置` 页面看就可以
