# 如果你什么都不懂，看这篇

这篇文档是写给普通使用者的。

如果你：

- 没装过 Python
- 没装过 Node.js
- 没装过 Git
- 平时只会下载文件、解压压缩包、双击程序

那就看这一篇，不用看别的。

## 你要不要安装 Python、Node.js、Git

不需要。

更直白地说：

- 不需要安装 Python
- 不需要安装 Node.js
- 不需要安装 Git

如果你拿到的是作者已经打包好的发布包，你不需要额外安装：

- Python
- Node.js
- Git

你只需要：

1. 下载作者发布好的 zip
2. 解压
3. 双击运行

## 这软件是干什么的

它是一个本地桌面软件，会在你的电脑上启动一个本地服务。

默认地址是：

```text
http://127.0.0.1:3011
```

它主要是把 `cursor2api` 做成一个更容易打开和管理的桌面程序。

你平时会接触到的功能主要是：

- 启动服务
- 停止服务
- 重启服务
- 开关 OCR
- 看当前状态
- 看统计
- 在浏览器打开原项目前端

## 第一次怎么用

### 第一步：下载

去作者给你的 GitHub Releases 下载最新 zip。

文件名通常会是：

```text
cursor2api-bridge-portable-win-x64.zip
```

### 第二步：解压

把 zip 解压到一个新的文件夹里。

注意：

- 不要直接在压缩包里双击运行
- 不要把新版本覆盖到旧版本目录上
- 最好放在你自己容易找到的位置，比如桌面或一个专门的软件文件夹

### 第三步：启动

解压后，找到：

```text
Cursor2API Bridge.exe
```

双击它。

第一次启动后，你会看到软件主界面。

## 软件里每个页面是干什么的

### 控制台

这里是最常用的页面。

你会看到：

- 顶部服务工作台
- 启动 / 停止 / 重启服务
- OCR 开关
- 最近标准输出
- 底部标准错误
- 本地地址 / 端口 / 模型的快速复制入口

如果你只是想“开起来用”，平时主要看这个页就够了。

### 设置

这里主要放：

- 浅色 / 深色 / 跟随系统
- 详细接入信息
- 打开原项目前端
- 打开健康检查
- 打开安装目录
- 打开数据目录
- 检查更新

### 统计

这里会看：

- 请求数
- 输入 Token
- 输出 Token
- 按模型统计

## 怎么知道它有没有正常运行

看“控制台”页面顶部的状态条。

如果状态是运行中，就说明本地服务已经起来了。

你也可以在浏览器里打开：

```text
http://127.0.0.1:3011/health
```

如果能看到正常返回，就说明服务活着。

## 怎么更新

### 最简单的方法：软件内更新

软件内更新依赖发布者上传好的这两个文件：

```text
cursor2api-bridge-portable-win-x64.zip
cursor2api-bridge-portable-win-x64.zip.sha256
```

1. 打开软件
2. 进入 `设置`
3. 点击 `检查更新`
4. 有新版本后点击 `下载并解压`
5. 下载完成后点击 `启动新版本`
6. 新版本会自动带上旧版本的 `data/`

这就是最推荐的更新方式。

### 手动更新

如果软件内更新暂时不能用，也可以手动更新：

1. 关闭旧版本软件
2. 去 GitHub Releases 下载最新 zip
3. 解压到一个新的文件夹
4. 如果你想保留配置、日志和数据库，就把旧目录里的 `data/` 复制到新目录

## 为什么不要覆盖旧目录

因为旧目录就是你的回滚点。

如果新版本有问题，你只要回去打开旧目录里的那个 `exe` 就行。

如果你直接覆盖旧目录，就没有这个退路了。

## 哪些文件是你的数据，哪些可以删

最重要的是：

```text
data/
```

这里通常会放：

- 配置
- 日志
- 数据库
- 软件自己的状态文件

如果你想保留自己的使用记录，最重要的就是保住 `data/`。

这些位置也值得知道：

- `data/logs/`：日志和数据库
- `updates/`：下载更新时用到的目录
- `source/`：随包带的源码快照

## 遇到问题先看哪里

### 情况 1：软件打不开或服务起不来

先看：

- 控制台底部的“标准错误”
- 控制台顶部的错误摘要

### 情况 2：更新按钮点了没反应

先看是不是提示：

```text
未配置更新源
```

这代表发布者还没有把 GitHub Releases 更新地址写进这个版本里。

这种情况下，你可以直接去 GitHub Releases 手动下载新 zip。

### 情况 3：打开新版本后东西都没了

先检查旧目录里的：

```text
data/
```

是不是没有复制过去。

### 情况 4：你担心隐私

要知道这软件默认会把一部分请求和响应信息写到本地日志里。

如果你处理的是敏感内容，可以先清理：

```text
data/logs/
```
