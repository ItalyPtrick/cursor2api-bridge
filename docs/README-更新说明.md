# Cursor2API Bridge 更新说明

## 给朋友的最短说明

软件内更新依赖下面这两个发布资产：

- `cursor2api-bridge-portable-win-x64.zip`
- `cursor2api-bridge-portable-win-x64.zip.sha256`

### 软件内更新

1. 打开旧版本，进入 `设置`
2. 点击 `检查更新`
3. 发现新版本后点击 `下载并解压`
4. 下载完成后点击 `启动新版本`
5. 新版本会自动带上旧版本的 `data/`

### 手动更新

1. 关闭旧版本托盘程序
2. 下载最新 zip
3. 解压到新的文件夹，不要覆盖旧目录
4. 如需保留配置、日志和数据库，把旧目录里的 `data/` 复制到新目录

## 为什么不原地覆盖

- 运行中的 `exe` 不适合直接覆盖
- 保留旧版本目录可以随时回滚
- 配置、日志、数据库都集中在 `data/` 目录里，迁移更直观

## 更新前建议

- 先在软件里停止服务，或直接从托盘退出程序
- 如涉及敏感内容，可先备份或清理 `data/logs/`
- 如需完整备份，直接复制整个 `data/` 目录即可

## 更新后检查

- 首页版本号是否更新
- 服务状态是否恢复为“运行中”
- `3011/health` 是否正常
- `/vuelogs` 或 `/logs` 是否能正常打开

## 常见问题

- 如果首页提示“未配置更新源”，说明打包者还没有把 GitHub Releases 仓库地址写进去
- 如果服务无法启动，先看首页里的“最近错误”
- 如果只想保留配置和日志，复制 `data/` 即可，不需要整目录搬运
